# Структура файлов
```txt
project_root/
├── CMakeLists.txt                 # Основной файл сборки
├── README.md
├── config/
│   └── default.yaml               # Параметры по умолчанию
├── src/
│   ├── core/
│   │   ├── ngram_model.h/c        # Хранение и операции с n-граммами
│   │   ├── model_io.h/c           # Сохранение/загрузка (текст/бинар)
│   │   └── token.h/c              # Определение токена и базовые утилиты
│   ├── parsers/
│   │   ├── parser_interface.h     # Абстрактный класс парсера
│   │   ├── md_parser.h/c          # Парсер Markdown
│   │   ├── c_parser.h/c           # Парсер C (использует общий токенизатор)
│   │   └── tokenizer.h/c          # Лексер для C (можно вынести в отдельный модуль)
│   ├── generation/
│   │   ├── markov.h/c             # Генератор на основе модели
│   │   └── stopwords.h/c          # Фильтр стоп-слов
│   ├── interactive/
│   │   ├── interactive.h/c        # Интерактивный режим
│   │   └── memory.h/c             # Управление долговременной памятью
│   ├── utils/
│   │   ├── file_utils.h/c         # Работа с файлами и директориями
│   │   ├── string_utils.h/c       # Строковые утилиты
│   │   └── time_utils.h/c         # Временные метки
│   └── main.c                     # Точка входа
├── tests/
│   ├── test_ngram.c
│   ├── test_parser_mock.c
│   ├── test_markov.c
│   └── ...
├── data/
│   ├── input/                     # Входные файлы для обучения
│   └── output/                    # Сгенерированные таблицы и результаты
├── memory/                        # Файлы долговременной памяти
└── scripts/
    ├── build.sh                   # Unix-скрипт сборки
    └── run.ps1                    # Windows-скрипт (обновлённый)
```

# UML-диаграммы потока данных
## Диаграмма компонентов (уровень модулей)
```txt
+-------------------+      +---------------------+      +-------------------+
|  Input files      |----->|    Parser Layer     |----->|  Model Layer      |
| (.md, .c, .h)     |      | - MdParser          |      | - NgramModel      |
+-------------------+      | - CParser           |      | - ModelIO         |
                           | - Tokenizer         |      +-------------------+
                           +---------------------+             |
                                    |                          |
                                    | (обучение)               | (сохранение/загрузка)
                                    v                          v
                           +---------------------+      +-------------------+
                           |   Output files      |<---- |  Generation Layer |
                           | (таблицы вероятн.)  |      | - MarkovGenerator |
                           +---------------------+      | - StopwordsFilter |
                                                       +-------------------+
                                                              |
                                                              | (генерация)
                                                              v
                                                       +-------------------+
                                                       | Interactive Shell |
                                                       | - Dialog manager  |
                                                       | - Memory storage  |
                                                       +-------------------+
```

## Диаграмма последовательности (генерация текста)
```txt
Пользователь  →  main  →  markov_load_tables()  →  ModelIO (чтение файлов)
                                                      ↓
                                                   NgramModel (заполнение)
                                                      ↓
Пользователь  →  markov_generate_ex()  →  MarkovGenerator
                                               │
                                               ├─ (если order=2) найти состояние триграммы
                                               │   └→ получить список переходов
                                               ├─ (если order>=1) найти состояние биграммы
                                               │   └→ получить список переходов
                                               └─ (иначе) найти униграмму
                                                   └→ получить список переходов
                                               │
                                               ├─ select_next() с учётом температуры
                                               │   └→ взвешенная случайная выборка
                                               ├─ (если фильтр стоп-слов) повторить выбор
                                               └─ вернуть следующий токен
                                                      ↓
                                                   буфер результата
                                                      ↓
Пользователь  ←  main (вывод в консоль/файл)
```

## Диаграмма состояний для обучения
```txt
[Начало] → Загрузка файлов из input/
          ↓
    Для каждого файла:
          ↓
    Определение типа (по расширению)
          ↓
    Парсинг → последовательность токенов
          ↓
    Для каждого токена: обновление уни-, би-, триграмм
          ↓
    Сохранение локальной таблицы (опционально)
          ↓
[Конец] → Запись глобальных таблиц вероятностей в output/
```
