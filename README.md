## Полная инструкция по компиляции и запуску программы

### 1. Структура проекта

Перед компиляцией убедитесь, что у вас есть следующая структура файлов (все файлы лежат в одной директории, например, `C:\Project\AID\`):

```
C:\Project\AID\
├── src\ (или текущая папка)
│   ├── ngram_model.h
│   ├── ngram_model.c
│   ├── tokenizer.h
│   ├── tokenizer.c
│   ├── c_parser.h
│   ├── c_parser.c
│   ├── md_parser.h
│   ├── md_parser.c
│   ├── markov.h
│   ├── markov.c
│   ├── interactive.h
│   ├── interactive.c
│   ├── main.c
│   └── (опционально) Makefile
├── input\
│   ├── *.md (для режима markdown)
│   └── *.c *.h (для режима code)
├── output\ (создаётся автоматически)
├── stopwords.txt (опционально, для фильтрации стоп-слов)
└── (исполняемый файл) markov_processor.exe
```

Если вы используете отдельную папку `src`, то при компиляции нужно указывать пути.

### 2. Компиляция

Используйте компилятор GCC (MinGW в Windows, или обычный gcc в Linux/macOS). Необходимо скомпилировать все `.c` файлы вместе:

```bash
gcc -std=c99 -Wall -Wextra -O2 -o markov_processor main.c ngram_model.c tokenizer.c c_parser.c md_parser.c markov.c interactive.c -lm
```

**Пояснение флагов:**
- `-std=c99` – стандарт C99 (используется в коде).
- `-Wall -Wextra` – включить предупреждения (полезно для отладки).
- `-O2` – оптимизация (можно убрать для отладки).
- `-o markov_processor` – имя выходного исполняемого файла.
- `-lm` – линковка с математической библиотекой (используется `pow()` в генераторе).

Если у вас все файлы в одной папке, команда будет такой же. Если файлы разбросаны по подпапкам, укажите пути: `src/main.c src/ngram_model.c ...`.

**Для Windows (MinGW/MSYS2):** компиляция аналогична.

**Для ускорения компиляции** можно создать **Makefile**:

```makefile
CC = gcc
CFLAGS = -std=c99 -Wall -Wextra -O2
LDFLAGS = -lm
TARGET = markov_processor
SRCS = main.c ngram_model.c tokenizer.c c_parser.c md_parser.c markov.c interactive.c
OBJS = $(SRCS:.c=.o)

all: $(TARGET)

$(TARGET): $(OBJS)
	$(CC) $(CFLAGS) -o $@ $^ $(LDFLAGS)

%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

clean:
	rm -f $(OBJS) $(TARGET)

.PHONY: all clean
```

Затем просто выполните `make` в папке с Makefile.

### 3. Подготовка входных данных

#### Для режима Markdown (по умолчанию)
- Создайте папку `input` в корне проекта.
- Поместите туда любые `.md` файлы (можно использовать примеры из предыдущих ответов).
- При необходимости создайте файл `stopwords.txt` в корне (если планируете фильтровать стоп-слова). Формат: одно слово на строку.

#### Для режима C-кода (генерация кода)
- Поместите в `input` файлы с расширением `.c` и `.h`.
- Стоп-слова для кода обычно не используются, поэтому можно не создавать.

**Важно:** Папка `output` создаётся автоматически, в неё будут сохраняться таблицы и сгенерированные файлы.

### 4. Запуск программы

Программа поддерживает несколько режимов через аргументы командной строки.

#### Базовый запуск (по умолчанию: парсинг Markdown, без генерации)
```bash
./markov_processor
```
Этот режим:
- Парсит все `.md` файлы из `input/`,
- Создаёт для каждого `.txt` файл в `output/` с частотами униграмм,
- Создаёт глобальные таблицы `global_unigram.txt`, `global_bigram.txt`, `global_trigram.txt` в `output/`.

#### Запуск с генерацией текста после парсинга
```bash
./markov_processor --generate
```
Добавляет генерацию одного блока текста с параметрами по умолчанию (порядок=1, T=0.8, 30 токенов, старт `<BOS>`). Результат сохраняется в `output/generated_output.txt`.

#### Запуск в интерактивном режиме (диалог)
```bash
./markov_processor --interactive
```
После парсинга и загрузки таблиц запускается диалог, где вы можете вводить фразы, а программа генерирует продолжение. Диалог сохраняется в `dialog_<timestamp>.md` (или в `dialog.md`, если задан параметр).

#### Совместное использование:
```bash
./markov_processor --generate --interactive
```

#### Выбор порядка, температуры и других параметров
```bash
./markov_processor --generate --order 2 --temp 0.9 --tokens 50 --start "Once"
```
- `--order 2` – использовать триграммы (0 – униграммы, 1 – биграммы, 2 – триграммы).
- `--temp 0.9` – температура (чем выше, тем случайнее).
- `--tokens 50` – максимальное число генерируемых токенов.
- `--start "Once"` – начальное слово.

#### Режим парсинга C-кода
```bash
./markov_processor --mode code
```
Парсит `.c` и `.h` файлы из `input/`, создаёт глобальные таблицы для кода.

#### Полный список опций:
```
--mode markdown|code    (режим парсинга)
--generate              (генерировать текст/код после парсинга)
--interactive           (запустить интерактивный диалог)
--order N               (0,1,2)
--temp T                (температура)
--tokens N              (максимум токенов)
--start "token"         (начальный токен)
--stopwords file        (путь к файлу стоп-слов)
--help                  (показать справку)
```

### 5. Примеры использования

**Пример 1:** Парсим Markdown, затем запускаем диалог с биграммами и температурой 0.8.

```bash
./markov_processor --interactive --order 1 --temp 0.8
```
В диалоге вы вводите текст, программа отвечает.

**Пример 2:** Парсим код и генерируем функцию на C.

```bash
./markov_processor --mode code --generate --order 2 --tokens 100 --start "int"
```
Сгенерированный код появится в `output/generated_output.txt`.

**Пример 3:** Только парсинг (без генерации), чтобы получить таблицы.

```bash
./markov_processor
```

### 6. Что появляется в папке `output` после работы

- `*.txt` (для каждого `.md` или `.c/.h` файла) – таблицы частот униграмм для этого файла.
- `global_unigram.txt` – вероятности переходов для униграмм (со сглаживанием).
- `global_bigram.txt` – вероятности для биграмм.
- `global_trigram.txt` – вероятности для триграмм.
- `generated_output.txt` – если использовался флаг `--generate`.
- `dialog_*.md` или `dialog.md` – если был интерактивный режим.
- `*.dot` – файлы графов (если экспорт включён, но в текущей версии он опционален).

### 7. Возможные ошибки и их решение

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `fopen: No such file or directory` | Папка `input` не существует или пуста. | Создайте папку `input` и поместите в неё файлы. |
| `error: unknown type name 'unigram_entry'` | Не подключён `ngram_model.h` или структуры определены позже. | Убедитесь, что все заголовочные файлы включены правильно. Проверьте порядок компиляции. |
| `undefined reference to 'pow'` | Не добавлена библиотека `-lm` | Добавьте `-lm` в конец команды компиляции. |
| `Error loading tables` | Файлы таблиц не найдены в `output/` после парсинга. | Сначала запустите программу без `--generate` или `--interactive`, чтобы создать таблицы. |
| `Permission denied` (Windows) | Исполняемый файл уже запущен. | Закройте предыдущий экземпляр программы. |
| Неправильная кодировка вывода в консоли (Windows) | Консоль в кодировке CP866, а программа выводит UTF-8. | Используйте команду `chcp 65001` перед запуском, либо установите шрифт, поддерживающий UTF-8. |

### 8. Дополнительные советы

- Для больших корпусов текста увеличьте `MAX_TOKEN_LEN` и размер хеш-таблиц в `ngram_model.h` при необходимости.
- Если вы хотите использовать свои стоп-слова, создайте файл `stopwords.txt` в корневой папке.
- Для визуализации графов установите Graphviz и выполните: `dot -Tpng output/unigram_graph.dot -o output/unigram_graph.png` (если генерация DOT включена).
- Для отладки добавьте флаг `-g` при компиляции и используйте `gdb`.

Теперь вы можете компилировать и запускать программу, экспериментировать с параметрами и генерировать текст или код на основе обученных моделей!
