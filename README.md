# AI-клон на цепях Маркова (Aid)
Проект реализует консольное приложение, генерирующее текст на основе биграммной модели Маркова, обученной на корпусе `.md`‑файлов. Все вычисления выполняются с использованием целочисленной арифметики и базовых алгоритмов (LFSR, бинарный поиск, разреженные матрицы) без привлечения готовых библиотек машинного обучения.
Код написан на C11 и ассемблере (Intel‑синтаксис, Microsoft x64 calling convention) с оглядкой на MISRA C:2012 – отсутствуют макросы, динамическая память после загрузки модели не выделяется, все управляющие конструкции снабжены фигурными скобками.

## Структура проекта
```text
Aid/
├── CMakeLists.txt          # Сценарий сборки CMake
├── asm_funcs.S             # Ассемблерные реализации LFSR и бинарного поиска
├── model_builder.c         # Утилита построения модели (model.bin + vocab.txt)
├── clone.c                 # Основное приложение – генератор текста
├── check_clone.gdb         # GDB‑скрипт автоматического тестирования
├── build_and_run.bat       # Скрипт полного цикла: сборка, тесты, запуск
├── corpus/                 # Папка с обучающими .md файлами (создаётся скриптом)
│   ├── science.md
│   └── questions.md
└── output/                 # Выходные артефакты (создаётся при сборке)
    ├── model.bin
    ├── vocab.txt
    └── clone.log
```

## Описание файлов
### Файл `CMakeLists.txt`
Сценарий сборки для CMake (версия ≥ 3.10).
- Определяет проект `MarkovClone` на языках C и ASM.
- Устанавливает стандарт C11.
- Создаёт две исполняемые цели:
  - `clone` (из `clone.c` и `asm_funcs.S`)
  - `model_builder` (из `model_builder.c`).

### Файл `asm_funcs.S`
Ассемблерный модуль, содержащий три функции, критические для производительности:
- `lfsr_rand` – 16‑битный генератор псевдослучайных чисел на основе 32‑битного LFSR (xorshift).
- `lfsr_seed` – инициализация состояния генератора.
- `binary_search_entry` – бинарный поиск в строке матрицы переходов. Возвращает идентификатор следующего токена по случайному числу и массиву накопленных вероятностей (cumulative probabilities).
  
Все функции следуют соглашению Microsoft x64: параметры передаются в `rcx`, `rdx`, `r8`, результат в `rax`.

### Файл `model_builder.c`
Консольная утилита для построения модели из корпуса.
Назначение: читает все `.md`‑файлы из заданной директории, токенизирует текст (разделители: пробелы, знаки препинания), формирует словарь уникальных слов, подсчитывает частоты биграмм и сохраняет результат в два файла:
- `model.bin` – бинарный файл, содержащий:
  - размер словаря (uint32_t),
  - количество ненулевых элементов матрицы,
  - смещения строк (uint32_t[]),
  - массив записей `{next_id (uint16_t), cum_prob (uint16_t)}` для всех строк.
- `vocab.txt` – текстовый файл словаря в кодировке UTF‑8 с BOM, по одному слову на строку.
Интерфейс: model_builder <input_dir> [output_dir]. Если выходная папка не указана, используется output.

### Файл `clone.c`
Основное консольное приложение, реализующее генератор текста.
Функциональность:
- Загружает модель (`model.bin`) и словарь (`vocab.txt`).
- Строит индекс для бинарного поиска по словарю.
- Переводит консоль в режим UTF‑8 (Windows).
- В интерактивном цикле принимает строку от пользователя, токенизирует её, использует последний токен как начальное состояние и генерирует продолжение (до 200 токенов или появления служебного токена `</S>`).
- Поддерживает флаги:
  - `--debug` – запись подробного лога в `clone.log`.
  - `--test` – запуск встроенных модульных тестов (LFSR, бинарный поиск, работа со словарём).

Безопасность: после загрузки модели динамическая память больше не выделяется. Все временные буферы фиксированного размера на стеке.

### Файл `check_clone.gdb`
Скрипт для отладчика GDB, автоматизирующий проверку работоспособности.

Выполняет два этапа:
- Модульные тесты – запуск `clone.exe --test`, анализ вывода на наличие строк `[PASS]`.
- Интеграционный тест – подача через `stdin` слова «Наука» и проверка, что в сгенерированном ответе присутствует это слово и отсутствует маркер `<UNK>`.
  
Запускается из `build_and_run.bat` после сборки.

### Файл `build_and_run.bat`
Пакетный файл для Windows, полностью автоматизирующий процесс от создания тестового корпуса до запуска приложения.

Шаги:
- Создаёт в папке `corpus` два `.md`‑файла в кодировке UTF‑8 через PowerShell.
- Принудительно завершает возможные висящие процессы `clone.exe` и `model_builder.exe`.
- Запускает CMake и Ninja для сборки.
- Строит модель, сохраняя её в папку `output`.
- Готовит тестовый ввод и прогоняет GDB‑тесты.
- При успешном прохождении тестов открывает новое окно CMD с запущенным клоном (режим `--debug`).

### Папки `corpus/` и `output/`
- `corpus/` – содержит обучающие текстовые файлы в формате Markdown. Создаётся автоматически скриптом `build_and_run.bat`.
- `output/` – содержит результаты работы утилиты `model_builder` (`model.bin`, `vocab.txt`) и лог‑файл `clone.log` (при использовании флага `--debug`). Также сюда помещаются временные файлы для GDB‑тестов.

## Сборка и запуск
Требования:
- Windows с установленным MSYS2 (или MinGW‑w64), CMake, Ninja, GDB.
- Компилятор GCC с поддержкой C11 и ассемблера.

Инструкция:
- Клонируйте или скопируйте все файлы проекта в одну директорию.
- Откройте командную строку и выполните `build_and_run.bat`.
- Дождитесь завершения сборки и тестов. Откроется новое окно с приглашением >.
- Введите фразу на русском языке (например, «Наука») и нажмите Enter. Программа сгенерирует продолжение.
- Для выхода введите exit.

## Примечания
- Код максимально приближен к MISRA C:2012 (обязательная инициализация переменных, запрет макросов, явные проверки возвращаемых значений, единый выход из функций).
- Ассемблерные вставки изолированы в отдельном модуле и используют стандартное для Windows x64 соглашение о вызовах.
- Логирование (флаг `--debug`) пишется в `clone.log` в папке `output` и содержит информацию о каждом переходе: начальный токен, случайное число, выбранный следующий токен.
- Мораль отсутствует – генерируемый текст определяется исключительно статистикой обучающего корпуса.
